This exercise uses Node.js to generate and receive data in a MySQL database.



This is customer.js

After initiating the node app my typing "node Bamazon.js" into the command line we are given a list of all items for purchase.

Using the list of all items, the user is prompted to first select an item by its ID for purchase. Then select the quantity. Finally you specify your spending limit. Submitting this returns a before-and-after shot of that item to verify.

After purchase, you're left given an updated full list of items, the item you bought, and how much it cost you. You'e also given the remaining amount of money you have as well as a list of all items in your cart.

As you buy more items, your cart will continue to grow until you run out of money or you choose to quit.

If you try to buy too many,an error is thrown explaining why you cannot.

This is manager.js

After initalizing BamazonManager.js in node by typing node BamazonManager into the command line, you are given a set of options. Clicking option 1 displays a general inventory of all stock.

Option 2 gives you a minified inventory of all items with less than 5 items. Option 3 will allow you to add items to the inventory.

As you see, when we add items to the inventory with option 3, we no longer see them displayed with option 2.

Option 4 allows you add a new item into inventory. After filling out the appropriate information, you can see it displayed within inventory.This item will now be treated the same as all other items.

As you can see, the new item is in the inventory and ready for purchase.
